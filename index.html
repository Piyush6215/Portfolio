First Project 2008 to 2009: Defence Accounts Pension, Allahabad – Defence Pension Management System

Designed and implemented a centralized SQL Server database for the Defence Accounts Pension Office, Allahabad, to manage pension records for retired defence personnel. The project involved collecting pension data from multiple teams, performing data cleansing and validation, developing ETL processes, and maintaining a secure, high-quality database to support accurate pension processing and reporting.

Key Responsibilities

  • Designed and implemented the SQL Server database schema for storing defence pension records.
  • Created normalized database tables, indexes, constraints, stored procedures, views, and user-defined functions.
  • Coordinated with multiple teams to collect and consolidate pension data from different sources.
  • Imported pension data received in CSV format into SQL Server using ETL processes.
  • Developed SQL scripts and ETL workflows for automated data loading and validation.
  • Performed data cleansing activities before loading records into the production database, including:
    • Removal of duplicate records
    • Trimming extra spaces from text fields
    • Removing unwanted special characters
    • Standardizing data formats
    • Validating mandatory fields
    • Verifying pension identifiers and record consistency
  • Implemented business validation rules to ensure data accuracy and integrity.
  • Created exception reports for rejected or invalid records to support issue resolution.
  • Optimized SQL queries and indexes for improved database performance.
  • Managed database backup, recovery, and routine maintenance activities.
  • Supported users by resolving data-related issues and ensuring timely availability of pension records.
  • Prepared operational and management reports for stakeholders.

Key Achievements

  • Successfully designed and implemented a centralized pension database for retired defence personnel.
  • Automated CSV-based data import and validation processes, reducing manual effort and improving accuracy.
  • Improved overall data quality by implementing duplicate detection, data cleansing, and validation rules.
  • Enhanced database performance through query optimization and index tuning.
  • Established reliable backup and recovery procedures to ensure data availability and business continuity.
  • Delivered a secure, scalable, and high-performance database solution supporting efficient pension record management.

Project 2 2010 to 2012: Sarva Shiksha Abhiyan & Saakshar Bharat Mission, Rajasthan – Government Survey Data Processing & Database Management System

Role: SQL Server DBA | ETL Developer | ABBYY ICR Developer
Client: Government of Rajasthan

Designed and implemented an end-to-end data capture and processing solution for the Government of Rajasthan to digitize large-scale survey forms and generate accurate demographic information. The system was developed to capture, validate, process, and store household and citizen data used for government planning, literacy analysis, and administrative reporting across Sarva Shiksha Abhiyan (SSA) and Saakshar Bharat Mission (SBM).

Key Responsibilities

  • Designed ICR-enabled survey forms for large-scale government data collection.
  • Developed ABBYY FlexiCapture templates and custom scripts for automated extraction of handwritten and printed information from ICR forms.
  • Configured field recognition rules, validation logic, and export settings in ABBYY FlexiCapture.
  • Conducted training programs for government officials, district coordinators, and field staff across all 33 districts of Rajasthan on the correct process for completing ICR forms.
  • Coordinated with district-level government offices throughout project implementation and deployment.
  • Extracted structured data from scanned forms and exported it into CSV format.
  • Developed SSIS packages to import extracted data into SQL Server.
  • Performed data cleansing, transformation, validation, and quality checks before loading records into the production database.
  • Implemented business validation rules to remove duplicate records, trim extra spaces, eliminate invalid special characters, and validate mandatory fields.
  • Designed and implemented the SQL Server database based on client requirements, creating tables, indexes, stored procedures, views, and database constraints.
  • Generated validation reports, exception reports, and reconciliation reports.
  • Supported User Acceptance Testing (UAT), production deployment, and final data handover to the Government of Rajasthan.

Government Data Processing Scope

  • Literacy status of individuals along with male and female literacy statistics.
  • Identification of female heads of households.
  • Family demographic information and household member details.
  • Village, Gram Panchayat, Block, and District-wise geo-records.
  • Population and family statistics for government survey and census-style reporting.
  • District-wise and state-level consolidated reports for planning and policy making.

Key Achievements

  • Successfully implemented an end-to-end ICR-based government survey data processing solution across all 33 districts of Rajasthan.
  • Automated data extraction from thousands of handwritten forms using ABBYY FlexiCapture and custom scripting.
  • Developed core SQL Server databases and automated robust ETL workflows using SSIS.
  • Significantly improved data quality through deep validation, duplicate detection, and automated cleansing processes.

  • Successfully completed production deployment and secured final data handover to the Government of Rajasthan.
